## 3. VALIDATION DURING BLOCK PRODUCTION: NO UTXO CHECK

**File:** src/blockchain.rs, lines 2618-2708

**CRITICAL GAP:** When producing a block, the code:
1. Gets finalized transactions
2. Checks for duplicates and double-spends (line 2625-2678)

When a transaction is RECEIVED, it's validated (pseudo-code):

            Some(UTXOState::SpentPending { txid, .. }) if txid == our_txid => {}  // OK  
            Some(UTXOState::SpentFinalized { txid, .. }) if txid == our_txid => {}  // OK

## 4. clearstucktransactions RPC: CHANGES UTXO STATE GLOBALLY

**File:** src/rpc/handler.rs, lines 3681-3750

\\\ust
async fn clear_stuck_transactions(&self) -> Result<Value, RpcError> {
    let finalized_txs = self.consensus.get_finalized_transactions_for_block();
    
    for tx in &finalized_txs {
        // Restore input UTXOs: SpentFinalized → Unspent
        for input in &tx.inputs {
            let outpoint = &input.previous_output;
            if matches!(
                self.utxo_manager.get_state(outpoint),
                Some(
                    crate::types::UTXOState::SpentFinalized { .. }
                        | crate::types::UTXOState::SpentPending { .. }
                        | crate::types::UTXOState::Locked { .. }
                )
            ) {
                self.utxo_manager
                    .update_state(outpoint, crate::types::UTXOState::Unspent);  // <-- UTXO STATE CHANGED!
                inputs_restored += 1;
            }
        }
        
        // Remove output UTXOs that were created by auto-finalization
        for (idx, _output) in tx.outputs.iter().enumerate() {
            let outpoint = OutPoint { txid, vout: idx as u32 };
            if let Err(e) = self.utxo_manager.remove_utxo(&outpoint).await {
                tracing::warn!("Failed to remove output UTXO {}", outpoint);
            } else {
                outputs_removed += 1;
            }
        }
    }
    
    // Clear from pools
    let txids: Vec<Hash256> = finalized_txs.iter().map(|tx| tx.txid()).collect();
    self.consensus.clear_finalized_txs(&txids);
}
\\\

**WHAT THIS DOES:**
1. Takes ALL finalized transactions from the pool
2. Changes their input UTXOs from SpentFinalized → Unspent
3. Removes their output UTXOs (which were created during auto-finalization)
4. Clears the transactions from the mempool

**THE SPLIT-BRAIN PROBLEM:**
- Some nodes run clearstucktransactions → UTXOs back to Unspent
- Other nodes DON'T run it → UTXOs still in SpentFinalized
- Now the network has **divergent UTXO state**
- Transactions can't be re-included because their inputs are in wrong state

- If clearstucktransactions was run on **node A**, its UTXOs are back to Unspent
- But **node B** (where TX was first finalized) still has the TX in finalized pool
- Node B's re-broadcast task sends TransactionFinalized to node A
- Node A receives it, **but cannot accept it** because:

  - The outputs no longer exist (were removed by clearstucktransactions)
  - **The transaction is incoherent with current UTXO state**

## ROOT CAUSE SUMMARY

### The Bug Chain:

1. **No UTXO re-validation during block production**
   - Finalized TXs fetched without checking if inputs still exist/are in correct state
   - Transactions silently skipped if they can't be included
   - Mempool not updated when transactions are skipped

2. **Split-brain on clearstucktransactions**
   - Some nodes clear UTXOs → Unspent
   - Other nodes don't → still SpentFinalized
   - Network diverges on UTXO state
   - Can't re-broadcast because state is incoherent

3. **No staleness or TTL on finalized transactions**
   - Transaction_pool has NO timeout mechanism
   - Transactions can sit in finalized pool forever
   - No automatic cleanup or detection of stuck state

4. **Re-broadcast sends to incompatible state**
   - Cleared nodes reject re-broadcast (wrong UTXO state)
   - Original nodes keep re-broadcasting forever
   - Creates infinite re-broadcast loop

---

## WHY HEIGHT 14879 STALLED

When clearstucktransactions was run:
1. Some nodes cleared finalized TXs, reset UTXO states
2. Those nodes can now produce blocks (clean state)
3. Other nodes still have finalized TXs stuck with SpentFinalized UTXOs
4. Those nodes:
   - Try to produce blocks from finalized pool
   - Every TX gets skipped (inputs in SpentFinalized, but no matching outputs)
   - Can't build valid blocks
   - Network stalls

---

## SOLUTIONS NEEDED

1. **During block production:**
   - Call alidate_transaction() on each finalized TX before including
   - If validation fails, REMOVE from finalized pool (don't just skip)
   - Log skipped transactions with reason

2. **clearstucktransactions:**
   - Must be coordinated across all nodes (use governance vote)
   - Or make it idempotent (safe to call multiple times)
   - Should sync UTXO state broadcast to all peers

3. **Add TTL to finalized transactions:**
   - If not included in a block after N minutes, auto-remove
   - Prevent unbounded accumulation

4. **Safe re-broadcast:**
   - Validate TX state matches current UTXO state before rebroadcasting
   - Don't rebroadcast if inputs don't exist
